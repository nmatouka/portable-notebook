Carrel
offline · open standards · no install for the runtime

Open notebooks by double-click — fully offline.

Carrel is a desktop app that opens marimo notebooks as live, interactive documents — no terminal, no hosting, no internet. The runtime lives in the app; a notebook file is just the portable payload.

▶ Try a live notebook View source on GitHub

The demo runs entirely in your browser via WebAssembly — the same engine the desktop app makes work offline.

What it is

An author bundles a notebook as a single .mnote file. A recipient installs the player once, then double-clicks any .mnote and it opens live and offline — sliders, inputs, and recomputation all work. It's the same bargain as Wolfram's old CDF Player, but built on open standards (HTML / WebAssembly / Python via Pyodide) instead of a proprietary engine.

How it works

  1. Double-click a .mnote (a marimo .py, optionally zipped with its wheels).
  2. The app serves its embedded WASM runtime over a custom secure protocol and injects the notebook's code.
  3. Pyodide boots and runs the notebook — with no network. Sliders and inputs recompute live.
  4. The webview is locked down: a notebook cannot phone home (strict CSP); only the app may fetch a missing package, once, with your OK.
